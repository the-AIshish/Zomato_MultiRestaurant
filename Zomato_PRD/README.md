# Zomato: Multi-Restaurant Single Order Experience

This folder contains the Strategic Lean PRD for the Zomato multi-restaurant checkout experience. This project explores how to reduce friction for group and family ordering by consolidating multiple restaurant carts into a single transaction.

## 📌 Project Overview

The "Multi-Restaurant Single Order" feature addresses a significant gap in the current food delivery user journey. Currently, users ordering for a household or office must place separate orders, pay multiple delivery fees, and track fragmented deliveries.

This proposal aims to:

- Increase Average Order Value (AOV) by encouraging cross-restaurant additions.
- Reduce Friction through a unified, single-payment checkout.
- Differentiate Zomato with a unique logistics solution for group dining.

## 📂 Document Focus: The "Lean" PRD

This specific document is a Strategic Lean PRD. It is designed for high-level stakeholder alignment and executive buy-in. Unlike a technical specification, it prioritizes:

- The "Why": Clearly defining the business objective and user pain points.
- MVP Guardrails: Identifying what is essential for launch vs. what belongs in Phase 2.
- Success Metrics: Establishing North Star KPIs like conversion lift and delivery SLA compliance.

## 🛠️ Product Management Skills Demonstrated

- **Strategic Prioritization**: Defining clear In-Scope vs. Out-of-Scope boundaries for an MVP.
- **User-Centric Design**: Mapping the transition from separate checkouts to a consolidated flow.
- **Risk Mitigation**: Identifying complexities in multi-merchant settlement and delivery routing early.
